# MDX pipeline spike

**Labs** are for experiments: cheap to try, easy to discard, honest about uncertainty.

## Setup (conceptual)

1. Store `.mdx` under `src/content/<domain>/`.
2. At build time (or on demand), read files and parse **frontmatter** + **body**.
3. Validate with **schemas** per domain; fail the build on invalid content.

## What we are not proving yet

- WYSIWYG editing
- Real-time collaboration
- Asset pipelines beyond static imports
